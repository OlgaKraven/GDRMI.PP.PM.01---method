## Задание

Требуется разработать информационную систему для сопровождения игрового продукта для изучения сквозного контура системы: архитектура → модель данных → жанровая адаптация → backend → API → интеграция клиента. Студент должен спроектировать архитектурную схему (Client → API → Backend → DB) с выделением ролей компонентов и описать один полный сценарий запроса по слоям (JSON → проверки → бизнес-логика → БД → ответ). Далее требуется реализовать универсальное ядро модели данных (Users, Roles, UserRoles, GameSessions, GameEvents, PlayerProgress, StatisticsDaily, LeaderboardScores) и предоставить корректный DDL-скрипт MySQL с PK/FK/UNIQUE/CHECK и JSON для событий. На основе выбранного жанра необходимо определить единицу игрового процесса, добавить 3–5 жанровых таблиц и сформировать минимум 6 событий с примерами payload. Backend должен включать middleware (requestId, authJwt, validate, errorHandler) и содержать хотя бы одну транзакционную операцию для итогового события (например, match_finish), которая атомарно обновляет событие, прогресс, статистику и лидерборд. API должно реализовывать минимум пять endpoint’ов (register, login, profile, events, leaderboard) с единым JSON-контрактом ответов. Клиент (Unity или Web) обязан реализовать login с сохранением JWT, загрузку профиля, отправку события с UUID и обработку ошибок 401 и 429. Завершающим этапом является тестирование: выполнение минимум восьми тестов (позитивных и негативных), проверка записей в MySQL, а также обязательная проверка дедупликации событий по eventId.
